~~ wackpackr ~~

"Wackpackr can't pack for shit!"

So what's this all about?

wackpackr accepts any kind of file, and will try compressing it with naive implementations of (1) Huffman and (2) LZSS algorithms.

Immediately after this, wackpackr decompresses the compressed binary, just to verify whether it can be returned back to its exact initial form.

The results are shown below for each algorithm.

Note that most files not only cannot be packed smaller, but on the contrary will take up even MORE space after compression. This is because many file types are compressed as-is with sophisticated techniques, so trying to squeeze them any further with a crappy application such as wackpackr is bound to backfire miserably.

If you want to see nice compression rates, text files usually do the trick. Long, uncompressed text with a lot of recurring patterns can at best be packed to 20% of its original size.

Note also that file size is capped currently at 1,024 kB.
